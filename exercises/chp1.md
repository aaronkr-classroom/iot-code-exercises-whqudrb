# chp1 연습문제

1.Q1
  - (a) 디바이스의 센서와 액추에이터의 역할을 비교하여 설명하세요.
    -  센서와 액추에이터는 디바이스에서 핵심적인 구성 요소로, 각각 입력과 출력을 담당하며 상호 보완적으로 작동한다. 센서는 외부 환경의 물리적 변화를 감지하여 이를 전기 신호로 변환하는 장치이다. 온도, 빛, 압력, 습도와 같은 다양한 물리량을 측정하고, 측정된 데이터를 시스템으로 전달하는 역할을하는 입력 장치이다. 액추에이터는 시스템으로부터 전달받은 전기 신호를 바탕으로물리적인 동작이나 변화를 만들어내는 장치이다. 예를 들어 모터의 회전, LED의 점등, 스피커의 소리 출력 등을 수행하는 출력장치이다.
  - (b) 디바이스와 대시보드에서 동일한 프로그래밍 언어를 사용하지 않는 이유를 설명하세요.
    -  디바이스는 센서와 액추에이터를 제어하는 역할을 하며, 일반적으로 마이크로컨트롤러와 같은 제한된 하드웨어 환경에서 동작하므로 실행 속도가 빠르고 하드웨어 제어에 적합한 C, C++과 같은 언어가 주로 사용된다. 대시보드는 사용자에게 데이터를 시각적으로 보여주고 제어할 수 있는 인터페이스를 제공하는 역할을 하므로 래픽 처리, 데이터 시각화, 네트워크 통신 등에 유리한 JavaScript, Python같은 언어가 사용된다. 또 디바이스는 임베디드 환경에서 실행되는 반면 대시보드는 웹이나 서버에서 실행되므로 주로 사용하는 언어가 다르다. 
  - (c) 브로커와 같은 정보 전달 전담 서버의 이점을 설명하세요.
    - 브로커를 사용하면 디바이스와 대시보드가 서로 직접 연결되지 않고 브로커를 통해 통신하게 되어 전체 시스템의 유연성과 확장성이 향상된다. 또 브로커는 메시지를 일시적으로 저장하고 관리하는 기능을 제공할 수 있어, 네트워크 장애나 일시적인 연결 끊김 상황에서도 데이터 유실을 줄일 수 있다. 여러 디바이스와 사용자(대시보드)가 동시에 데이터를 주고받을 경우, 브로커가 이를 중재하여 효율적으로 분배한다. 특히 발행/구독(Publish/Subscribe) 방식에서는 하나의 데이터를 여러 수신자에게 쉽게 전달할 수 있다. 또한 확장과 유지보수에 유리하다.
2. Q2
  - (c) Mosquitto와 Node-RED에 대해 설명하고, 실행에 적합한 컴퓨팅 환경을 제시하시오.
    - Mosquitto는 경량 메시지 프로토콜인 MQTT(Message Queuing Telemetry Transport)를 기반으로 하는 브로커 프로그램이다. Node-RED는 흐름 기반(Flow-based) 프로그래밍 도구로, 데이터를 시각적으로 연결하여 애플리케이션을 구성할 수 있는 플랫폼이다. 두 시스템은 함께 사용될 때 IoT 환경에서 효율적인 데이터 통신과 관리가 가능하며, 라즈베리파이 또는 리눅스 서버와 같은 환경에서 실행하는 것이 적합하다.
